# MVP Scope

## Build Type

Premium one-page landing page with lightweight discovery-call intake flow.

## Not Building Yet

- Full client portal
- User accounts
- Custom backend
- Payment processing
- Deposit checkout
- Proposal generator
- CRM dashboard
- Newsletter system
- AI chatbot
- Blog
- Course platform
- Membership area

## MVP Sections

1. Hero
2. Problem
3. Somatic Strategy
4. Somatic Strategy Arc
5. Services
6. Who This Is For
7. Social Proof
8. About Lee
9. Discovery Call CTA
10. Footer

## Primary CTA

Book a Discovery Call

## Functional MVP

The site should allow a qualified buyer to:

1. Understand the offer
2. Understand who it is for
3. See proof of credibility
4. Read about Lee
5. Request or book a discovery call
6. Provide enough intake information for Lee to prepare

## Intake Fields

- Name
- Organization
- Role/title
- Email
- Phone
- Organization type
- Team/cohort size
- Service interest
- Desired timeline
- Location
- In-person, virtual, or hybrid
- What prompted you to reach out?
- What would success look like?
- Has budget been approved?
- Estimated budget range
- Are you the decision-maker?
- Anything Lee should know before the call?

## Calendar Protection Rules To Confirm

- Minimum notice
- Weekly cutoff
- Discovery call windows
- Max calls per day
- Max calls per week
- Buffer between calls
- Same-week approval policy
- Rescheduling policy
- Cancellation policy

## MVP Stack Recommendation

- Vite
- React
- Tailwind
- Cloudflare Pages
- Cloudflare DNS
- Tally, Formspree, or similar form tool
- Calendly or Acuity for scheduling
- Stripe or Square later, after service rules are defined

## Launch Definition

The MVP is ready when:

- The page looks premium on desktop and mobile
- CTA works
- Intake path works
- Contact info is visible
- No fake proof is included
- No broken links exist
- Email records are not harmed
- Lee can review it without needing to understand the tech
